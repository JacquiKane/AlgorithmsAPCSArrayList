How Efficient Is Binary Search?

We say of the order of, designated by O(...), to describe the worst case in terms of comparisons. This is termed big O representation. The best case is described using big Omega notation, Ω(...).

The array has to be sorted before binary search can be used. This may take away from any efficiency gains! In the best case, the item is found right away, after a single split. In this case, the best case scenario is described as big Omega of 1, Ω(1). In the worst case, we have to keep splitting the list, and end up with approximately log n comparisons. This is designated using big O notation, as O(log n).
