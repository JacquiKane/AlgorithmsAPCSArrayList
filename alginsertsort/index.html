Sort ArrayList Using Insertion Sort

Iterate through the ArrayList, inserting each element in the correct order.

Let's use an example. Below, the ArrayList is called nums. The ArrayList elements are shifted as insertions occur, following the insert sort algorithm.


Sample Code, Assuming Array Of Ints - nums

            ArrayList<Integer> nums = new ArrayList<Integer>();
            // Add elements to the ArrayList
            // ....
            for (int index = 1; index < nums.size(); index++) { 
                Integer currentElement = nums.get(index);
                     int j = index - 1; 
                       
                     while (j >= 0 && ((nums.get(j).compareTo(currentElement)) > 0)){ 
                         nums.set(j + 1, nums.get(j)); 
                         j = j - 1; 
                         } 
                         nums.set(j+1, currentElement);
                     } 
                     

                
                
** public int compareTo(Integer anotherInt)
Parameter : anotherInt, the Integer to be compared.
Return :
- This method returns the value 0 if this Integer is equal to the argument Integer 
- a value less than 0 if this Integer is numerically less than the argument Integer 
- a value greater than 0 if this Integer is numerically greater than the argument Integer

How Efficient Is Insert Sort?

If there are n items in the array, then the best case scenario is when the array is already sorted - in this case we compare n elements but do not have to shift them. The worst case scenario is when the array is sorted in reverse order. Given this situation, we have n2 comparisons.

We say of the order of, designated by O(...), to describe the worst case. This is termed big O representation. For insert sort, the worst case is O(n2) comparisons.

The best case is an array that is sorted. This is described using big Omega notation, Ω(...), as Ω(n) comparisons.
