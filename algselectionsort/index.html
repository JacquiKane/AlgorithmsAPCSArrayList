Sort ArrayList Instance Using Selection Sort

This sorting algorithm relies on swapping assumed minimum values with actual minimum values in an ArrayList, always adding to the end of a sorted list of items. Let's look at a pseudo-code first, and assume we have an ArrayList instance of n integers, nums. 

                        for each element, a, in the ArrayList, assume this element is the minimum
                            for each element, b, in the rest of the ArrayList
                              compare element b with the assumed minimum
                              if the element is less than the assumed minimum, 
                                then set the assumed minimum to be this element
                        swap a and the minimum

This sort is ascending, the same principles apply to descending sort, but we look for maximum values instead of minimum values.


Sample Code, Assuming ArrayList Of Integers - nums


    for (int index = 0; index < nums.size(); index++) { 
        int min_idx = index; 
        // Inner loop of selection sort runs once, to n-1
        for (int j = index+1; j < nums.size(); j++) 
            if ((nums.get(j).compareTo(nums.get(min_idx)) < 0)) 
                min_idx = j; 
                
        // Swap found minimum element with first element
        int temp = nums.get(min_idx); 
        nums.set(min_idx, nums.get(index)); 
        nums.set(index, temp);                 
        }

                
                
** public int compareTo(Integer anotherInt)
Parameter : anotherInt, the Integer to be compared.
Return :
- This method returns the value 0 if this Integer is equal to the argument Integer 
- a value less than 0 if this Integer is numerically less than the argument Integer 
- a value greater than 0 if this Integer is numerically greater than the argument Integer

You can implement method compareTo, in your own classes, so that ArrayList 
instances containing instances of these classes can use algorithms like this one, and 
compare objects based on your criteria.

How Efficient Is Selection Sort?

If there are n items in the array, we compare n elements, with n elements. We have n2 comparisons.

We say of the order of, designated by O(...), to describe the worst case. This is termed big O representation. For selection sort, even if the array is sorted we still have n2 comparisons. The best case is described using big Omega notation, Ω(...), as Ω(n2) comparisons.

Selection Sort has the same worst case as best case! Ω(n2) and O(n2)
